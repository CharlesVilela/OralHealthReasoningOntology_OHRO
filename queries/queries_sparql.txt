# 8. Consultas SPARQL (Competency Questions)

A ontologia foi validada por meio de consultas SPARQL baseadas em **Competency Questions (CQs)**, que ajudam a verificar se o modelo ontológico responde adequadamente às perguntas para as quais foi projetado. As consultas abaixo exploram taxonomia, relações clínicas, fatores de risco, diagnósticos e estrutura de datasets. 

---

## CQ1 — Quais são as principais categorias de lesões orais representadas na ontologia?

Objetivo: recuperar todas as subclasses da classe `Lesion`.

```sparql
PREFIX ex: <http://example.org/oral-cancer-ontology#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?category ?label
WHERE {
  ?category rdfs:subClassOf ex:Lesion .
  OPTIONAL { ?category rdfs:label ?label . }
}
ORDER BY ?category
```

## CQ2 — Quais fatores de risco aparecem em pacientes que possuem determinada lesão?

Objetivo: identificar correlação entre diagnósticos clínicos e fatores de risco.

```sparql
PREFIX owl: <http://www.w3.org/2002/07/owl#>
PREFIX oral: <http://example.org/oral-cancer-ontology#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

SELECT DISTINCT ?lesionType ?riskFactorType
WHERE {
  ?image oral:hasClinicalDiagnosis ?lesion ;
         oral:belongsToPatient ?patient .
  ?patient oral:hasRiskFactor ?riskFactor .
  ?lesion rdf:type ?lesionType .
  ?riskFactor rdf:type ?riskFactorType .
  FILTER(?lesionType != owl:NamedIndividual)
  FILTER(?riskFactorType != owl:NamedIndividual)
}
ORDER BY ?lesionType ?riskFactorType
```

## CQ5 — Qual o perfil clínico e hábitos de cada paciente?

Objetivo: combinar dados demográficos e fatores de risco.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>

SELECT ?patient ?id ?age ?gender
       (GROUP_CONCAT(DISTINCT STRAFTER(STR(?riskFactor),"#"); separator=", ") AS ?riskFactors)

  ?patient a oral:Patient .
  OPTIONAL { ?patient oral:patientID ?id . }
  OPTIONAL { ?patient oral:age ?age . }
  OPTIONAL { ?patient oral:gender ?gender . }
  OPTIONAL { ?patient oral:hasRiskFactor ?riskFactor . }
}
GROUP BY ?patient ?id ?age ?gender
ORDER BY ?patient
```

## CQ6 — Quais anotações de imagem (lesões) estão associadas a cada exame?

**Objetivo:** recuperar as anotações das imagens, incluindo a lesão associada e sua bounding box.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT
    ?patient
    ?image
    ?annotation
    (COALESCE(?label, STRAFTER(STR(?lesion),"#")) AS ?lesionName)
    ?x
    ?y
    ?width
    ?height
    ?area
WHERE {
    ?image oral:hasAnnotation ?annotation ;
           oral:belongsToPatient ?patient .

    OPTIONAL {
        ?annotation oral:indicatesLesion ?lesion .
        OPTIONAL { ?lesion rdfs:label ?label . }
    }

    OPTIONAL {
        ?annotation oral:hasBoundingBox ?bbox .
        OPTIONAL { ?bbox oral:bb_x ?x . }
        OPTIONAL { ?bbox oral:bb_y ?y . }
        OPTIONAL { ?bbox oral:bb_width ?width . }
        OPTIONAL { ?bbox oral:bb_height ?height . }
        OPTIONAL { ?bbox oral:area ?area . }
    }
}
ORDER BY ?patient ?image ?annotation
```

**Interpretação:**
Permite localizar semanticamente cada lesão dentro da imagem, servindo como ponte entre a ontologia e tarefas de visão computacional (segmentação/detecção).

## CQ7 — Quais pacientes apresentam mais de um tipo de lesão diagnosticada?

**Objetivo:** identificar pacientes com múltiplas categorias clínicas.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>

SELECT
    ?patient
    (COUNT(DISTINCT ?category) AS ?numCategories)
    (GROUP_CONCAT(DISTINCT STRAFTER(STR(?category),"#"); separator=", ") AS ?categories)
WHERE {
    ?image oral:belongsToPatient ?patient ;
           oral:hasCategory ?category .
}
GROUP BY ?patient
HAVING (COUNT(DISTINCT ?category) > 1)
ORDER BY DESC(?numCategories)
```

**Interpretação:**
Identifica pacientes com múltiplas condições ou progressão clínica entre categorias diagnósticas.

## CQ8 — Quais fatores de risco estão mais frequentemente associados a lesões malignas?

**Objetivo:** correlacionar fatores de risco com malignidade.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

SELECT
    ?lesionType
    ?riskFactorType
    (COUNT(DISTINCT ?patient) AS ?frequency)
WHERE {
    ?image oral:belongsToPatient ?patient ;
           oral:hasClinicalDiagnosis ?lesion .

    ?patient oral:hasRiskFactor ?riskFactor .

    ?lesion rdf:type ?lesionType .
    ?riskFactor rdf:type ?riskFactorType .

    FILTER(?lesionType != oral:Lesion)
}
GROUP BY ?lesionType ?riskFactorType
ORDER BY DESC(?frequency)
```

**Interpretação:**
Permite identificar padrões epidemiológicos relevantes, como forte associação entre tabagismo e OSCC.

## CQ9 — Quais imagens estão associadas a pacientes com diagnóstico confirmado de câncer oral?

**Objetivo:** filtrar imagens de casos confirmados de câncer oral.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT
    ?patient
    ?image
    ?lesion
    ?label
WHERE {
    ?image oral:belongsToPatient ?patient ;
           oral:hasClinicalDiagnosis ?lesion .

    ?lesion rdfs:label ?label .

    FILTER(
        REGEX(
            LCASE(STR(?label)),
            "oral cancer|cancer|carcinoma"
        )
    )
}
ORDER BY ?patient
```

**Interpretação:**
Útil para seleção de datasets específicos para treinamento de modelos de IA focados em câncer oral.

## CQ10 — Como as categorias do dataset (Normal, Benign, OPMD e Malignant) se relacionam com os diagnósticos clínicos das lesões?

**Objetivo:** validar consistência entre labels computacionais e diagnósticos médicos.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

SELECT
    ?category
    ?diagnosisType
    (COUNT(*) AS ?frequency)
WHERE {
    ?image oral:hasCategory ?category ;
           oral:hasClinicalDiagnosis ?diagnosis .

    ?diagnosis rdf:type ?diagnosisType .

    FILTER(?diagnosisType != oral:Lesion)
}
GROUP BY ?category ?diagnosisType
ORDER BY ?category DESC(?frequency)
```

**Interpretação:**
Ajuda a verificar inconsistências entre classificação computacional e diagnóstico clínico.

## CQ11 — Quais pacientes têm fatores de risco, mas não têm diagnóstico registrado?

**Objetivo:** identificar pacientes que necessitam acompanhamento.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>

SELECT
    ?patient
    ?patientID
    (GROUP_CONCAT(DISTINCT STRAFTER(STR(?riskFactor),"#"); separator=", ") AS ?riskFactors)
WHERE {
    ?patient a oral:Patient ;
             oral:hasRiskFactor ?riskFactor .

    OPTIONAL {
        ?patient oral:patientID ?patientID .
    }

    FILTER NOT EXISTS {
        ?patient oral:hasMedicalImage ?image .
        ?image oral:hasClinicalDiagnosis ?diagnosis .
    }
}
GROUP BY ?patient ?patientID
ORDER BY ?patientID
```

**Interpretação:**
Importante para triagem preventiva e rastreamento clínico.

## CQ12 — Quais diagnósticos estão mais frequentemente associados a um determinado fator de risco?

**Objetivo:** descobrir associações entre hábitos e diagnósticos.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

SELECT
    (STRAFTER(STR(?riskFactorType),"#") AS ?RiskFactor)
    (STRAFTER(STR(?diagnosisType),"#") AS ?Diagnosis)
    (COUNT(DISTINCT ?patient) AS ?Patients)
WHERE {
    ?patient oral:hasRiskFactor ?riskFactor ;
             oral:hasMedicalImage ?image .

    ?image oral:hasClinicalDiagnosis ?diagnosis .

    ?riskFactor rdf:type ?riskFactorType .
    ?diagnosis rdf:type ?diagnosisType .

    FILTER(?diagnosisType != oral:Lesion)
}
GROUP BY ?riskFactorType ?diagnosisType
ORDER BY DESC(?Patients)
```

**Interpretação:**
Suporta análises clínicas para investigação de fatores etiológicos.

## CQ14 — Quais pacientes apresentam múltiplos fatores de risco?

**Objetivo:** identificar pacientes com maior vulnerabilidade clínica.

```sparql
PREFIX oral: <http://example.org/oral-cancer-ontology#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

SELECT
    ?patientID
    (COUNT(DISTINCT ?riskFactor) AS ?numRisks)
    (GROUP_CONCAT(
        DISTINCT STRAFTER(STR(?riskFactorType),"#");
        separator=", "
    ) AS ?riskFactors)
WHERE {
    ?patient oral:hasRiskFactor ?riskFactor ;
             oral:patientID ?patientID .

    ?riskFactor rdf:type ?riskFactorType .
}
GROUP BY ?patientID
HAVING (COUNT(DISTINCT ?riskFactor) > 1)
ORDER BY DESC(?numRisks)
```

**Interpretação:**
Auxilia na priorização de pacientes para monitoramento e intervenções preventivas.
